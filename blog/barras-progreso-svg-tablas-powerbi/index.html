Barras de progreso con SVG para tablas de Power BI

Crearemos barras de progreso con marcador de porcentajes utilizando archivos SVG en tablas de Power BI para mejorar la visualización de datos.

Publicado: 10 Set. 2025
Categorías:

Power BI

Publicado por:

Junior Salvador

Contexto

En esta publicación exploraremos el uso de archivos SVG (Gráficos Vectoriales Escalables) para la creación de barras de progreso en tablas de Power BI. Gracias a la flexibilidad del formato SVG, es posible editar el código que lo conforma (de manera similar a las etiquetas HTML) y así dinamizar su contenido, posición, tamaño, color, entre otros atributos. Esto permite generar visualizaciones más atractivas y personalizadas.

El propósito es optimizar la presentación de los datos en las tablas, facilitando su interpretación y logrando un impacto visual más destacado. A continuación, se describen los pasos para implementar esta técnica en Power BI.

Instrucciones

A. Descargar plantilla .pbix

Para facilitar la comprensión de este artículo, he preparado una plantilla de Power BI (.pbix) que contiene todos los elementos necesarios para seguir los pasos descritos en la publicación. Descarga el archivo plantillas_barras_de_progreso.pbix ( aquí ) que tiene 4 versiones de barras de progreso a implementar. Evidentemente, puedes crear tus propios diseños SVG, pero para efectos prácticos y de simplicidad, usaré los que he diseñado y compartido en la plantilla.

B. Implementación
  1. Al abrir la plantilla notaremos que las barras de progreso están establecidas en medidas, las cuales son: BarraProgresoNro1, BarraProgresoNro2, BarraProgresoNro3 y BarraProgresoNro4. Copia el código DAX de la medida de la barra de progreso que desees usar, luego crea una nueva medida en tu informe y pega el código.
  1. Edita la medida para que se ajuste a tus necesidades. Solo debes modificar el valor asignado a la variable ValorPorcentaje —modifica tabla_empleados[Avance] por la tabla y columna que requieras. Considera que los valores de la columna Avance deben estar entre 0 y 100, adáptalos a tu contexto. 

								VAR ValorPorcentaje = MAX(tabla_empleados[Avance])
							

								Avance = 
									SWITCH(
										TRUE(),
										tabla_empleados[IdEmpleado] = 1, 30,
										tabla_empleados[IdEmpleado] = 2, 100,
										tabla_empleados[IdEmpleado] = 3, 45,
										tabla_empleados[IdEmpleado] = 4, 80,
										tabla_empleados[IdEmpleado] = 5, 28,
										tabla_empleados[IdEmpleado] = 6, 10,
										tabla_empleados[IdEmpleado] = 7, 36,
										tabla_empleados[IdEmpleado] = 8, 61,
										tabla_empleados[IdEmpleado] = 9, 70,
										tabla_empleados[IdEmpleado] = 10, 50,
										BLANK()
									)
  1. Una vez creada la medida, es importante establecer su clasificación como Dirección URL de la imagen en la opción "Categoría de datos" del menú "Herramientas de medición", sino verás el código SVG en el objeto visual en lugar de su representación gráfica.
  1. Usa la medida como columna en un objeto de tabla.
  2. Finalmente, con el objeto visual seleccionado ajusta la proporción del SVG en "Tamaño de la imagen" dentro de "Formato". Ajusta el alto y ancho hasta que la barra de progreso se vea correctamente. Por default, lo establecí en 42px de alto y 200px de ancho.

Opcional: ¿Puedo crear mis propios diseños SVG?

Recomiendo ampliamente el uso de Figma, la herramienta con la que he elaborado los cuatro diseños presentados anteriormente. Se trata de una plataforma en línea de diseño gráfico y prototipado muy intuitiva, que permite exportar los gráficos directamente en formato .SVG. Posteriormente, estos archivos pueden manipularse en alguno de los numerosos editores de código disponibles en línea para SVG o, como en mi caso, mediante Visual Studio Code junto con una extensión específica. A continuación, comparto los enlaces a dichas herramientas.

1. Creación de gráficos SVG: FIGMA

2. Editor de código SVG (online): SVGViewer

3. Editor de código (local): Visual Studio Code y extensión SVG by jock

Consideraciones al crear gráficos SVG para Power BI
  1. Lo más sencillo para abrir un archivo .SVG y ver su código es usar el Bloc de notas.
  2. ¿Se tiene que editar el código del .SVG exportado por FIGMA? Sí. Es indispensable reemplazar las comillas dobles ("") por comillas simples ('') en TODO el código. Si no lo haces, obtendrás un error en Power BI. Para las 4 versiones que he compartido, ya he realizado este cambio, así que no debes preocuparte por ello, pero si decides crear tus propios diseños, recuerda hacer este ajuste.

✅ Correcto: Uso de comillas simples 


								<svg width='270' height='57' viewBox='0 0 200 57' fill='none' xmlns='http://www.w3.org/2000/svg'>
									... el código de tu gráfico SVG ...
								</svg>

❌ Incorrecto: Uso de comillas dobles 


								<svg width="270" height="57" viewBox="0 0 200 57" fill="none" xmlns="http://www.w3.org/2000/svg">
									... el código de tu gráfico SVG ...
								</svg>
  1. Si vas a dinamizar texto (como en nuestro caso), no recomiendo crear los textos del SVG en ningún editor visual, mejor usa la etiqueta <text> directamente en el código. El motivo es porque los editores visuales suelen tratar el texto como una forma compleja por tener muchas curvas y lo crean codificando una etiqueta <path> con el atributo d= en lugar de un texto simple.
